2、header标签

header标签在语法规范中包含标题、标题介绍或标题导航。 header不允许嵌套，不允许出现在地址（address）或页脚（footer）内。 *注：header应在CSS中被定义成display:block;其他结构性元素(article、footer等)也应如此。 这样可以保证不兼容新标签的浏览器也能正确显示这些元素。 其他HTML5新元素的编码都已经有通过JavaScript库的实现方案

3、hgroup

<hgroup> 超过两个以上的h标签不再允许直接书写，将使用hgroup组织起来。 hgroup的总体起始位置取决于首个标题出现的位置，一般只包含h1~h6标签。 <h1>Welcome to HTML5</h1> h1元素在一个Web页面中原则上只能出现一次。 <h2>Excitement!</h2> </hgroup>

4、aside

aside元素作用于页面上一些与主题联系不大而相对独立的区域。 与article相对应：如果article是主干，则aside是分支。

<label>Username <input name="user" type="text" size="10"></label><br><br>使用label将Username和后面的input包裹起来。以便区分。 <label>Password <input name="pass" type="text" size="10"></label>

6、article

article对应Web页面中的一个独立区域，该区域包括新闻、博客文章之类的内容。 如果article是相互嵌套的，原则上来说，内嵌article应该与外部article有着相同的主题样式。 article用到display: block; 使内容宽度被缩减到环绕div的79%。 <article> <img src="文档结构和语义/img/3.png"/> <h2>这里是article</h2> <address>who you are</address> <time pubdate="pubdate" datetime="2016-04-15">星期五</time> <p><q>这是纯洁无暇的力量 </q>塔里克</p> </article>

7、address,time,q

<address>Samuel H, Library of GXU</address> address包含联系信息（邮件地址、名字、公司、地址等相关信息），如果需要单独表达地址时用<p>。 address作用于与其最近的article元素，若没有article元素则作用于整个文档。 address默认为斜体显示。 <time pubdate="pubdate" datetime="2016-02-25T19:45:00Z">Thursday, February 25th 2016 @19:45:00</time> pubdate 属性指示 <time> 元素中的日期 / 时间是文档（或最近的前辈 <article> 元素）的发布日期。 目前所有主流浏览器都不支持 <pubdate> 标签。 datetime 属性规定日期或时间。在该元素的内容中未指定日期或时间时，使用该属性。 目前所有主流浏览器都不支持 <datetime> 标签。 time根据其pubdate、datetime属性，共同形成文档的时间戳。 *注：time元素是方便供机器读取，而不是形成特殊的时间内容。 Web页面显示的仍然time内自定义的内容。 <p>This is the first HTML5 website that I made.</p> CSS中，p用到clear: both; 以消除img(float: left;)对其的影响。 <p><q>This is perfect!</q>Says Nick.</p> 引用部分用到q。英文引号标记"并非HTML语言的一部分。 但配合CSS伪元素:before和:after，浏览器仍能为q元素生成合适的样式。

8、figure

<figure> figure和figcaption不能算作结构性元素。 figure用于组织图片，figcaption则用于对这一组图片的描述。 每一个figure元素只能包括一个figcaption。 figcaption的位置在figure内是任意的。 但是在默认情况下，figcaption的大小与figure的大小一致。 <img src="img/3.png" alt="Circle1"> <img src="img/1.png" alt="Circle2"> <img src="img/2.png" alt="Circle3"> <figcaption>Bonne Nuit, The Monge!</figcaption> </figure>

9、footer

<footer class="shadow"> footer包含了许多信息，诸如作者、其他相关页面（friendlink）、注意事项（版权、免责声明）等。 <p class="copyright">&copy; 2016 <mark>GXU</mark></p> 一般版权用<p>写。 mark用于高亮标记一段文字。 <nav> *<h2>Navigation</h2> <div> <a href="about:blank" class="shadow">1</a> <a href="about:blank" class="shadow">2</a> <a href="about:blank" class="shadow">3</a> <a href="about:blank" class="shadow">4</a> </div> </nav> </footer>

9、大纲选择器

在aside和footer nav中都有一个隐藏的h2（*）。下面介绍大纲算法： HTML5大纲分析器会分析出下面的条目结构： 1.Welcome to HTML5! (h1) *1.Link Block (h2) 1.Questionnare (h3) 2.Login (h3) 3.Quick Links (h3) 2.DELL (h2) *3.Navigation (h2) 由此，HTML5大纲分析器会按层次分析h标签内容。如果忽略Link Block和Navigation， 则分析器会以Untitle Section替换。 合理使用h1~h6标签，既可以保持结构完整，也可以使大纲清晰易读。 要逐渐培养大纲思维，清晰划分Web页面结构。 同时，需要注意的是，在使用hgroup时，大纲仅包含hgroup中的最高级别。 目前还没有浏览器可以直接执行大纲算法，搜索引擎也没有改进机制。
